Production & client work

Production Monitoring Dashboard

Raspberry Pi • Linux • Flask • systemd

Problem: A production line ran with water-tank temperature and ozone concentration completely unmonitored — anomalies weren't caught until final test.

Built: A Raspberry Pi dashboard capturing both readings in real time, served over a local web server, auto-starting via systemd, accessible from anywhere on the floor.

Result: Process variables that were previously invisible are now watched continuously, and drift gets flagged before units reach end-of-line test.

OTA Firmware Update System

AWS • Raspberry Pi • Automatic rollback

Problem: Field instruments deployed at remote sites needed firmware updates without a technician visit — and a bad update could brick an instrument in the field.

Built: Over-the-air updates delivered through AWS, with automatic fallback to the last known-good image on any failed update.

Result: Rollback-safe remote updates for instruments in the wild; a failed push recovers itself instead of generating a service call.

Production AI Agent Deployment (Zeno)

Raspberry Pi 5 • Claude API • systemd • WireGuard

Problem: A client needed a recurring research workflow handled automatically, reliably, every day — without a server bill or public internet exposure.

Built: A hardened AI agent on a Raspberry Pi 5: systemd auto-restart, ARM64 memory tuning, VPN-only access, key-only SSH, and strict secret-file permissions.

Result: Running today under a paid managed-service contract — setup fee plus monthly retainer.

RISC-V Firmware Migration

Bare-metal C • RISC-V assembly • Cross-compilation

Problem: Legacy firmware for 5G/telecom timing-IC platforms had to move to a brand-new custom RISC-V architecture under tight memory and timing constraints.

Built: Ported and refactored the firmware, authored the Makefiles and linker scripts for the new toolchain, and validated peripheral behavior on target silicon.

Result: Multi-target builds on the new architecture with improved memory efficiency and execution speed.

Edge AI Model Deployment

NVIDIA Jetson Orin Nano • Hugging Face • Python

Problem: A field-instrument product needed on-device AI inference — no cloud dependency, no latency, no per-query cost.

Built: Locally trained and deployed a Hugging Face model on a Jetson Orin Nano and integrated inference into the instrument.

Result: AI features running entirely at the edge, validated on target hardware before shipment.
